API Reference
GET /health Liveness probe — returns 200 with env name, version, corpus size
POST /reset Start new episode. Body: {"task_id": "label_only"}
POST /step Submit action. Body: {"label": "urgent", "route": "engineering", "summary": "...", "reply": "..."}
POST /state Query full episode state. Body: {}
GET /tasks Enumerate all tasks with metadata, weights, and thresholds
GET /metrics Aggregate corpus stats + grader design rationale (for judges)

Quick start
# 1. Start the server
docker build -t email-triage-env .
docker run -p 7860:7860 email-triage-env

# 2. Reset and run an episode
curl -X POST http://localhost:7860/reset -H 'Content-Type: application/json' \
  -d '{"task_id": "label_only"}'

curl -X POST http://localhost:7860/step -H 'Content-Type: application/json' \
  -d '{"label": "urgent"}'

# 3. Run baseline inference
export HF_TOKEN=hf_your_token
python inference.py
